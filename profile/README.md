# Cobalt Truck Equipment – IT

## Overview
Welcome to the Cobalt Truck Equipment IT GitHub organization. This environment is used to manage, version, and maintain automation, integrations, and technical resources that support business operations and infrastructure.

Our goal is to provide a structured, secure, and scalable foundation for IT-driven solutions across the organization.

---

## Repository Structure

This organization is organized by functional domains to ensure clarity, maintainability, and proper access control:

### IT Automation
Repositories focused on scripting and operational tooling.
- PowerShell, Python, and administrative scripts
- Microsoft 365 and Azure automation
- Infrastructure and support utilities

### Power Platform
Repositories for Microsoft Power Platform solutions.
- Power Automate flows (exported JSON/ZIP)
- Power Apps
- Templates and environment configurations

### Enterprise Systems
Repositories supporting business-critical platforms and integrations.
- Salesforce and SAP resources
- API integrations and middleware
- System-specific documentation and tooling

---

## Standards & Practices

To ensure consistency and reliability across all repositories:

- All code and assets should be version-controlled
- Changes should be submitted via pull requests where applicable
- Scripts, flows, and integrations must be documented
- Naming conventions and folder structures should be followed
- Sensitive data (credentials, secrets, tokens) must never be stored in repositories

---

## Access & Contributions

Access to repositories is managed through GitHub teams based on job function and responsibility.

If you need access or would like to contribute:
- Contact the IT department
- Follow repository-specific contribution guidelines
- Submit changes using pull requests for review
