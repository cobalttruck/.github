# Security Policy

## Overview
This repository is maintained by the Cobalt Truck Equipment IT team and follows internal security and operational best practices.

---

## Reporting a Vulnerability

If you discover a security vulnerability or sensitive issue:

- Do NOT create a public issue or pull request
- Report it directly to the IT team

Contact:
- Cobalt IT Department
- support@cobalttruck.com

---

## Guidelines

When contributing to this repository:

- Do not include passwords, API keys, tokens, or credentials
- Do not commit environment-specific secrets or configuration files containing sensitive data
- Use secure methods for handling authentication (e.g., environment variables, secure vaults)

---

## Scope

This policy applies to all repositories within the Cobalt Truck Equipment GitHub organization.

---

## Best Practices

- Validate scripts and integrations before deploying to production
- Limit permissions to only what is necessary
- Regularly review and update automation and integrations for security risks

---

## Notes

Security is a shared responsibility. When in doubt, escalate concerns to the IT team before proceeding.
